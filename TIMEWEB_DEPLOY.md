# Инструкция по развертыванию на Timeweb Cloud (App Platform)

Для того чтобы приложение работало корректно и было доступно "в реальном мире", мы настроили сборку (Dockerfile и скрипты `package.json`). Следуйте этой простой инструкции для запуска на Timeweb Cloud Apps.

## Шаг 1. Подготовка репозитория (Экспорт)
1. В AI Studio скачайте проект (выберите "Download ZIP" или "Export to GitHub").
2. Лучший способ - выгрузить проект в свой репозиторий GitHub.
3. Убедитесь, что `package.json`, `server.ts`, а также `Dockerfile` лежат в корне репозитория.

## Шаг 2. Создание приложения в Timeweb Cloud
1. Перейдите в панель управления [Timeweb Cloud](https://timeweb.cloud/) и авторизуйтесь.
2. В левом меню выберите раздел **"Apps (Приложения)"** и нажмите **"Создать"**.
3. Выберите источник исходного кода: подключите ваш аккаунт **GitHub** и выберите репозиторий с выгруженным проектом.
4. Выберите ветку для деплоя (обычно `main` или `master`).

## Шаг 3. Настройка параметров деплоя (Критически важно)
При создании приложения укажите следующие параметры:
- **Окружение (Environment):** Docker (так как мы добавили Dockerfile, Timeweb соберет и запустит контейнер автоматически).
- **Порт:** Укажите порт `3000` (наше приложение по умолчанию слушает этот порт).
- **Команда запуска (Run command):** Можно оставить пустым, Dockerfile автоматически исполнит `npm start`.
- **Переменные окружения (Environment Variables):** Если вы используете Gemini API или другие ключи (например, `GEMINI_API_KEY`), укажите их в блоке переменных окружения на панели Timeweb.

## Шаг 4. Деплой
1. Нажмите "Запустить" и дождитесь окончания сборки.
2. Timeweb Cloud автоматически выдаст вам публичный домен (URL), по которому ваше приложение будет доступно всему интернету.
3. Вы также можете привязать свой собственный домен в настройках приложения в Timeweb.

## Как это работает под капотом (Full-stack)
Мы используем Express backend (`server.ts`) и клиентский интерфейс на React/Vite.
В процессе сборки контейнера (Docker) происходит:
1. `npm run build` - компилирует React-приложение в папку `dist` и собирает backend (`server.ts`) с помощью `esbuild` в единый файл `dist/server.cjs`.
2. Команда `npm start` (которая запускает `node dist/server.cjs`) поднимает рабочий production-сервер, который отдает скомпилированную статику и обслуживает API.

Всё готово для использования! 🚀
